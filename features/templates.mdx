A **template** is a reusable HTML design that can be used across campaigns and transactional messages. Templates typically have standard header and footer areas with logos and branding, where campaign content is inserted in the middle.

## Template Types

Listmonk supports three template types:

<CodeGroup>
```sql schema.sql
DROP TYPE IF EXISTS template_type CASCADE;
CREATE TYPE template_type AS ENUM ('campaign', 'campaign_visual', 'tx');
```
</CodeGroup>

| Type | Description |
|------|-------------|
| **campaign** | Used for email campaigns, wraps campaign content |
| **campaign_visual** | Used with the visual email builder |
| **tx** | Used for transactional messages sent via API |

## Go Template Syntax

Listmonk uses [Go templates](https://pkg.go.dev/text/template) for powerful, dynamic templating. Template expressions are written between double curly braces: `{{ }}` 

### Subscriber Fields

Access subscriber information in templates:

| Expression | Description |
|------------|-------------|
| `{{ .Subscriber.UUID }}` | Randomly generated unique ID |
| `{{ .Subscriber.Email }}` | Subscriber's email address |
| `{{ .Subscriber.Name }}` | Full name |
| `{{ .Subscriber.FirstName }}` | First name (auto-extracted) |
| `{{ .Subscriber.LastName }}` | Last name (auto-extracted) |
| `{{ .Subscriber.Status }}` | Status (enabled, disabled, blocklisted) |
| `{{ .Subscriber.Attribs }}` | JSON attributes map |
| `{{ .Subscriber.CreatedAt }}` | Creation timestamp |
| `{{ .Subscriber.UpdatedAt }}` | Last update timestamp |

### Campaign Fields

| Expression | Description |
|------------|-------------|
| `{{ .Campaign.UUID }}` | Unique campaign ID |
| `{{ .Campaign.Name }}` | Internal campaign name |
| `{{ .Campaign.Subject }}` | Email subject line |
| `{{ .Campaign.FromEmail }}` | Sender email address |

### Built-in Functions

| Function | Description | Example |
|----------|-------------|---------|
| `{{ Date "2006-01-01" }}` | Print current date in specified format | `{{ Date "2006-01-02" }}` |
| `{{ TrackLink "https://link.com" }}` | Generate tracking URL | `<a href="{{ TrackLink "https://example.com" }}">Click</a>` |
| `https://link.com@TrackLink` | Shorthand for TrackLink | `<a href="https://example.com@TrackLink">Link</a>` |
| `{{ TrackView }}` | Insert tracking pixel | Place once in template footer |
| `{{ UnsubscribeURL }}` | Unsubscribe/manage preferences URL | Use in footer |
| `{{ MessageURL }}` | URL to hosted message version | View in browser link |
| `{{ OptinURL }}` | Double opt-in confirmation URL | For opt-in campaigns |
| `{{ Safe "<!-- comment -->" }}` | Insert raw HTML without escaping | `{{ Safe .CustomHTML }}` |

## Template Structure

Every campaign template must include the content placeholder **exactly once**:

```html
{{ template "content" . }}
```

This is where the campaign body is inserted.

<Warning>
Templates without the `{{ template "content" . }}` placeholder will fail validation.
</Warning>

## Example Template

Here's a complete campaign template with header, footer, and content placeholder:

<CodeGroup>
```html template.html
<!DOCTYPE html>
<html>
  <head>
    <style>
      body {
        background: #eee;
        font-family: Arial, sans-serif;
        font-size: 16px;
        color: #111;
      }
      header {
        border-bottom: 1px solid #ddd;
        padding-bottom: 30px;
        margin-bottom: 30px;
      }
      .container {
        background: #fff;
        width: 450px;
        margin: 0 auto;
        padding: 30px;
      }
    </style>
  </head>
  <body>
    <section class="container">
      <header>
        <!-- Header appears in all emails -->
        Hi {{ .Subscriber.FirstName }}!
      </header>

      <!-- Campaign content is inserted here -->
      <div class="content">
        {{ template "content" . }}
      </div>

      <footer>
        Copyright 2024. All rights Reserved.
        <br>
        <a href="{{ UnsubscribeURL }}">Unsubscribe</a>
      </footer>

      <!-- Tracking pixel -->
      {{ TrackView }}
    </section>
  </body>
</html>
```
</CodeGroup>

<Info>
For plaintext campaigns, create a template with no HTML content and just the placeholder: `{{ template "content" . }}`
</Info>

## Example Campaign Body

Campaign bodies use the same template syntax and can access subscriber data:

<CodeGroup>
```html campaign-body.html
Hey, did you notice how the template showed your first name?
Your last name is {{ .Subscriber.LastName }}.

You have done {{ .Subscriber.Attribs.projects }} projects.

{{ if eq .Subscriber.Attribs.city "Bengaluru" }}
  You live in Bangalore!
{{ else }}
  Where do you live?
{{ end }}

Here is a link for you to click that will be tracked:
<a href="{{ TrackLink "https://google.com" }}">Google</a>
```
</CodeGroup>

This example uses:
- Subscriber attributes access (`.Subscriber.Attribs.projects`)
- Conditional logic (`if eq`)
- Link tracking (`TrackLink`)

## Accessing Nested Attributes

For nested JSON attributes, use dot notation:

```html
{{ .Subscriber.Attribs.city }}
{{ .Subscriber.Attribs.stack.preferred_language }}
```

Given this subscriber:

```json
{
  "city": "Bengaluru",
  "stack": {
    "preferred_language": "go",
    "frameworks": ["echo", "go"]
  }
}
```

## Sprig Functions

Listmonk integrates the [Sprig library](https://masterminds.github.io/sprig/) with 100+ utility functions:

### String Functions

```html
{{ upper .Subscriber.Name }}  <!-- JOHN DOE -->
{{ lower .Subscriber.Email }} <!-- john@example.com -->
{{ trim .Subscriber.Name }}   <!-- Remove whitespace -->
```

### Date Functions

```html
{{ now | date "2006-01-02" }}  <!-- Current date -->
{{ .Subscriber.CreatedAt | date "Jan 2, 2006" }}
```

### Default Values

```html
{{ .Subscriber.Attribs.city | default "Unknown" }}
```

### List Operations

```html
{{ range .Subscriber.Attribs.spoken_languages }}
  - {{ . }}
{{ end }}
```

<Warning>
Sprig functions are Turing-complete and can program complex behavior. Only give templating permissions to trusted users to avoid potential memory or performance issues.
</Warning>

## Template Schema

<CodeGroup>
```sql schema.sql
CREATE TABLE templates (
    id              SERIAL PRIMARY KEY,
    name            TEXT NOT NULL,
    type            template_type NOT NULL DEFAULT 'campaign',
    subject         TEXT NOT NULL,
    body            TEXT NOT NULL,
    body_source     TEXT NULL,
    is_default      BOOLEAN NOT NULL DEFAULT false,
    created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```
</CodeGroup>

### Key Fields

- **subject**: Only relevant for transactional templates (campaign subjects are per-campaign)
- **body**: Compiled HTML template
- **body_source**: Visual editor source (for visual templates)
- **is_default**: Only one template can be default

## Template Validation

Templates are validated on creation and update:

<CodeGroup>
```go cmd/templates.go
func (a *App) validateTemplate(o models.Template) error {
    if !strHasLen(o.Name, 1, stdInputMaxLen) {
        return errors.New(a.i18n.T("campaigns.fieldInvalidName"))
    }

    if o.Type == models.TemplateTypeCampaign && !regexpTplTag.MatchString(o.Body) {
        return echo.NewHTTPError(http.StatusBadRequest,
            a.i18n.Ts("templates.placeholderHelp", "placeholder", tplTag))
    }

    if o.Type == models.TemplateTypeTx && strings.TrimSpace(o.Subject) == "" {
        return echo.NewHTTPError(http.StatusBadRequest,
            a.i18n.Ts("globals.messages.missingFields", "name", "subject"))
    }

    return nil
}
```
</CodeGroup>

## System Templates

System templates render public pages and automated emails:

- **Public pages**: Subscription management, opt-in confirmation, archive pages
- **System emails**: Opt-in confirmation, campaign status notifications, data export emails

<Info>
System templates are bundled with listmonk but can be customized by copying the static directory and passing `--static-dir=your/custom/path` flag.
</Info>
