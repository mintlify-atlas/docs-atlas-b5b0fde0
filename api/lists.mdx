---
title: Lists API
description: API endpoints for managing mailing lists in Listmonk
---

Manage mailing lists including creating, updating, querying, and deleting lists. Lists can be public or private with single or double opt-in.

## Get Lists

<api-endpoint>
  <endpoint>
    <method>GET</method>
    <path>/api/lists</path>
  </endpoint>
</api-endpoint>

Retrieve all lists with optional filtering and pagination.

<Note>Lists with `status: archived` are hidden from list selectors in campaigns, public subscription forms, and roles by default. They can only be viewed by filtering with `status=archived` or by viewing all lists without a status filter.</Note>

### Query Parameters

<ParamField query="query" type="string">
  String for list name search.
</ParamField>

<ParamField query="status" type="string">
  Status to filter lists. Options: `active`, `archived`. Defaults to showing all lists if not specified.
</ParamField>

<ParamField query="minimal" type="boolean">
  If true, returns lists without subscriber counts (faster). Defaults to false.
</ParamField>

<ParamField query="tag" type="string[]">
  Tags to filter lists. Repeat in the query for multiple values.
</ParamField>

<ParamField query="order_by" type="string">
  Sort field. Options: `name`, `status`, `created_at`, `updated_at`.
</ParamField>

<ParamField query="order" type="string">
  Sorting order. Options: `ASC`, `DESC`.
</ParamField>

<ParamField query="page" type="number">
  Page number for pagination.
</ParamField>

<ParamField query="per_page" type="number">
  Results per page. Set to 'all' to return all results.
</ParamField>

### Response

<ResponseField name="data" type="object">
  <ResponseField name="results" type="array">
    Array of list objects.
    <ResponseField name="id" type="number">
      List's unique ID.
    </ResponseField>
    <ResponseField name="uuid" type="string">
      List's UUID.
    </ResponseField>
    <ResponseField name="name" type="string">
      List name.
    </ResponseField>
    <ResponseField name="type" type="string">
      List type: `public`, `private`.
    </ResponseField>
    <ResponseField name="optin" type="string">
      Opt-in type: `single`, `double`.
    </ResponseField>
    <ResponseField name="status" type="string">
      List status: `active`, `archived`.
    </ResponseField>
    <ResponseField name="tags" type="string[]">
      Tags associated with the list.
    </ResponseField>
    <ResponseField name="subscriber_count" type="number">
      Number of subscribers in the list.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      Timestamp when list was created.
    </ResponseField>
    <ResponseField name="updated_at" type="string">
      Timestamp when list was last updated.
    </ResponseField>
  </ResponseField>
  <ResponseField name="total" type="number">
    Total number of lists.
  </ResponseField>
  <ResponseField name="page" type="number">
    Current page number.
  </ResponseField>
  <ResponseField name="per_page" type="number">
    Results per page.
  </ResponseField>
</ResponseField>

<RequestExample>
```bash Get All Lists
curl -u "api_user:token" -X GET 'http://localhost:9000/api/lists?page=1&per_page=100'
```

```bash Get Active Lists
curl -u "api_user:token" -X GET 'http://localhost:9000/api/lists?status=active&per_page=100'
```

```bash Get Archived Lists
curl -u "api_user:token" -X GET 'http://localhost:9000/api/lists?status=archived&minimal=true&per_page=all'
```
</RequestExample>

## Get Public Lists

<api-endpoint>
  <endpoint>
    <method>GET</method>
    <path>/api/public/lists</path>
  </endpoint>
</api-endpoint>

Retrieve public lists with name and UUID to submit a subscription. This is an unauthenticated endpoint to enable scripting for subscription forms.

<Note>This endpoint only returns lists with `type: public` and `status: active`. Archived lists are never shown on public subscription forms.</Note>

<RequestExample>
```bash cURL
curl -X GET 'http://localhost:9000/api/public/lists'
```
</RequestExample>

<ResponseExample>
```json
[
  {
    "uuid": "55e243af-80c6-4169-8d7f-bc571e0269e9",
    "name": "Opt-in list"
  }
]
```
</ResponseExample>

## Delete Multiple Lists

<api-endpoint>
  <endpoint>
    <method>DELETE</method>
    <path>/api/lists</path>
  </endpoint>
</api-endpoint>

Delete multiple lists by IDs or by a search query.

<Note>Users can only delete lists they have `manage` permission for. Any lists in the query that the user doesn't have permission to manage is ignored.</Note>

### Query Parameters

<ParamField query="id" type="number[]">
  Required if `query` not provided. One or more list IDs to delete.
</ParamField>

<ParamField query="query" type="string">
  Required if `id` not provided. Search query to filter lists for deletion (same as the GET query).
</ParamField>

<RequestExample>
```bash Delete by IDs
curl -u "api_user:token" -X DELETE 'http://localhost:9000/api/lists?id=10&id=11&id=12'
```

```bash Delete by Query
curl -u "api_user:token" -X DELETE 'http://localhost:9000/api/lists?query=test%20list'
```
</RequestExample>

<ResponseExample>
```json
{
    "data": true
}
```
</ResponseExample>

## Error Responses

All endpoints return standard HTTP status codes:

- `200 OK` - Request successful
- `400 Bad Request` - Invalid parameters
- `401 Unauthorized` - Missing or invalid authentication
- `404 Not Found` - List not found
- `500 Internal Server Error` - Server error
