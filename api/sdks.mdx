## Python SDKs

**[listmonk](https://github.com/mikeckennedy/listmonk)**

Python API client for Listmonk.

```python
from listmonk import ListmonkAPI

client = ListmonkAPI("http://localhost:9000", "username", "password")
subscribers = client.get_subscribers()
```

**[listmonk-api](https://github.com/Knuckles-Team/listmonk-api)**

Another Python API client with comprehensive endpoint coverage.

```python
from listmonk_api import Listmonk

api = Listmonk(url="http://localhost:9000", username="user", password="pass")
api.create_subscriber(email="user@example.com", name="User")
```

## PHP SDKs

**[listmonk-php-client](https://github.com/arunnabraham/listmonk-php-client)**

PHP API client for Listmonk.

```php
use Listmonk\Client;

$client = new Client('http://localhost:9000', 'username', 'password');
$subscribers = $client->subscribers()->getAll();
```

**[php-listmonk](https://github.com/junisan/php-listmonk)**

Alternative PHP client library for the Listmonk API.

## JavaScript/TypeScript SDKs

**[listmonk-nodejs-api](https://github.com/mihairaulea/listmonk-nodejs-api)**

NodeJS API client for Listmonk.

```javascript
const Listmonk = require('listmonk-nodejs-api');

const client = new Listmonk('http://localhost:9000', 'username', 'password');
const subscribers = await client.getSubscribers();
```

**[@maloma/listmonk](https://codeberg.org/maloma/listmonk-sdk)**

NodeJS/Bun TypeScript SDK with full type definitions.

```typescript
import { ListmonkClient } from '@maloma/listmonk';

const client = new ListmonkClient({
  baseUrl: 'http://localhost:9000',
  username: 'user',
  password: 'pass'
});

const subscribers = await client.subscribers.list();
```

### Framework-Specific

**[listmonk-laravel](https://github.com/theafolayan/listmonk-laravel)**

Laravel package for Listmonk integration.

```php
use Listmonk\Facades\Listmonk;

Listmonk::createSubscriber([
    'email' => 'user@example.com',
    'name' => 'John Doe'
]);
```

**[nuxt-listmonk](https://github.com/roncallyt/nuxt-listmonk)**

Listmonk module for Nuxt.js applications.

```javascript
export default {
  modules: ['nuxt-listmonk'],
  listmonk: {
    baseURL: 'http://localhost:9000',
    username: 'user',
    password: 'pass'
  }
}
```

## Go SDK

**[go-listmonk](https://github.com/EzeXchange-API/go-listmonk)**

Go API client library for Listmonk.

```go
import "github.com/EzeXchange-API/go-listmonk"

client := listmonk.NewClient("http://localhost:9000", "username", "password")
subscribers, err := client.GetSubscribers()
```

## Crystal SDK

Crystal language API client for Listmonk.

```crystal

```

## API Authentication

All SDKs use HTTP Basic Authentication. Configure your client with:

```bash
Username: admin (or your configured username)
Password: your_api_password
```

## Contributing Your SDK

If you've created an SDK or integration for Listmonk:

1. Test thoroughly against the latest Listmonk version
2. Include documentation and examples
3. Add your project to the [Listmonk documentation repository](https://github.com/knadh/listmonk)
4. Share in the [Listmonk community](https://github.com/knadh/listmonk/discussions)

## SDK Requirements

When building an SDK, ensure it supports:

### Core Endpoints
- Subscribers management (CRUD)
- Lists management (CRUD)
- Campaigns management (CRUD)
- Templates management (CRUD)
- Transactional emails

### Authentication
- HTTP Basic Authentication
- Proper error handling for 401/403

### Best Practices
- Proper error handling and reporting
- Type safety (where applicable)
- Comprehensive documentation
- Example code
- Unit tests

### Nice to Have
- Async/await support
- Retry logic for failed requests
- Rate limiting awareness
- Webhook handling
- Batch operations
