---
title: Subscribers API
description: API endpoints for managing subscribers in Listmonk
---

Manage subscribers including creating, updating, querying, and deleting subscriber records. Subscribers can be assigned to lists, blocklisted, and have their bounce records managed.

## Get Subscribers

<api-endpoint>
  <endpoint>
    <method>GET</method>
    <path>/api/subscribers</path>
  </endpoint>
</api-endpoint>

Retrieve all subscribers with optional filtering and pagination.

### Query Parameters

<ParamField query="query" type="string">
  Subscriber search by SQL expression.
</ParamField>

<ParamField query="list_id" type="int[]">
  ID of lists to filter by. Repeat in the query for multiple values.
</ParamField>

<ParamField query="subscription_status" type="string">
  Subscription status to filter by if there are one or more `list_id`s.
</ParamField>

<ParamField query="order_by" type="string">
  Result sorting field. Options: `name`, `status`, `created_at`, `updated_at`.
</ParamField>

<ParamField query="order" type="string">
  Sorting order: `ASC` for ascending, `DESC` for descending.
</ParamField>

<ParamField query="page" type="number">
  Page number for paginated results.
</ParamField>

<ParamField query="per_page" type="number">
  Results per page. Set as 'all' for all results.
</ParamField>

### Response

<ResponseField name="data" type="object">
  <ResponseField name="results" type="array">
    Array of subscriber objects.
    <ResponseField name="id" type="number">
      Subscriber's unique ID.
    </ResponseField>
    <ResponseField name="uuid" type="string">
      Subscriber's UUID.
    </ResponseField>
    <ResponseField name="email" type="string">
      Subscriber's email address.
    </ResponseField>
    <ResponseField name="name" type="string">
      Subscriber's name.
    </ResponseField>
    <ResponseField name="status" type="string">
      Subscriber status: `enabled`, `blocklisted`.
    </ResponseField>
    <ResponseField name="attribs" type="object">
      JSON object with custom subscriber attributes.
    </ResponseField>
    <ResponseField name="lists" type="array">
      Array of lists the subscriber is subscribed to.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      Timestamp when subscriber was created.
    </ResponseField>
    <ResponseField name="updated_at" type="string">
      Timestamp when subscriber was last updated.
    </ResponseField>
  </ResponseField>
  <ResponseField name="total" type="number">
    Total number of subscribers.
  </ResponseField>
  <ResponseField name="page" type="number">
    Current page number.
  </ResponseField>
  <ResponseField name="per_page" type="number">
    Results per page.
  </ResponseField>
</ResponseField>

<RequestExample>
```bash cURL
curl -u 'api_username:access_token' 'http://localhost:9000/api/subscribers?page=1&per_page=100'
```

```bash Filter by Lists
curl -u 'api_username:access_token' 'http://localhost:9000/api/subscribers?list_id=1&list_id=2&page=1&per_page=100'
```

```bash SQL Query
curl -u 'api_username:access_token' -X GET 'http://localhost:9000/api/subscribers' \
    --url-query 'page=1' \
    --url-query 'per_page=100' \
    --url-query "query=subscribers.name LIKE 'Test%' AND subscribers.attribs->>'city' = 'Bengaluru'"
```
</RequestExample>

## Create Subscriber

<api-endpoint>
  <endpoint>
    <method>POST</method>
    <path>/api/subscribers</path>
  </endpoint>
</api-endpoint>

Create a new subscriber.

### Body Parameters

<ParamField body="email" type="string" required>
  Subscriber's email address.
</ParamField>

<ParamField body="name" type="string" required>
  Subscriber's name.
</ParamField>

<ParamField body="status" type="string" required>
  Subscriber's status: `enabled`, `blocklisted`.
</ParamField>

<ParamField body="lists" type="number[]">
  List of list IDs to subscribe to.
</ParamField>

<ParamField body="attribs" type="object">
  Optional JSON object attributes for the subscriber that can be used in message templates. Example: `{"location": "Somewhere"}`
</ParamField>

<ParamField body="preconfirm_subscriptions" type="boolean">
  If true, subscriptions are marked as confirmed and no opt-in emails are sent for double opt-in lists.
</ParamField>

<RequestExample>
```bash cURL
curl -u 'api_username:access_token' 'http://localhost:9000/api/subscribers' \
  -H 'Content-Type: application/json' \
  --data '{
    "email":"subscriber@domain.com",
    "name":"The Subscriber",
    "status":"enabled",
    "lists":[1],
    "attribs":{
      "city":"Bengaluru",
      "projects":3,
      "stack":{"languages":["go","python"]}
    }
  }'
```
</RequestExample>

<ResponseExample>
```json
{
  "data": {
    "id": 3,
    "created_at": "2019-07-03T12:17:29.735507+05:30",
    "updated_at": "2019-07-03T12:17:29.735507+05:30",
    "uuid": "eb420c55-4cfb-4972-92ba-c93c34ba475d",
    "email": "subscriber@domain.com",
    "name": "The Subscriber",
    "attribs": {
      "city": "Bengaluru",
      "projects": 3,
      "stack": { "languages": ["go", "python"] }
    },
```

```json
}
```
</ResponseExample>

## Public Subscription

<api-endpoint>
  <endpoint>
    <method>POST</method>
    <path>/api/public/subscription</path>
  </endpoint>
</api-endpoint>

Create a public subscription. This endpoint does not require authentication and accepts both form-encoded or JSON-encoded body.

### Body Parameters

<ParamField body="email" type="string" required>
  Subscriber's email address.
</ParamField>

<ParamField body="name" type="string">
  Subscriber's name.
</ParamField>

<ParamField body="list_uuids" type="string[]" required>
  List of list UUIDs.
</ParamField>

<RequestExample>
```bash JSON Request
curl 'http://localhost:9000/api/public/subscription' \
  -H 'Content-Type: application/json' \
  --data '{
    "email":"subscriber@domain.com",
    "name":"The Subscriber",
    "list_uuids": ["eb420c55-4cfb-4972-92ba-c93c34ba475d"]
  }'
```

```bash Form Request
curl 'http://localhost:9000/api/public/subscription' \
  -d 'email=subscriber@domain.com' \
  -d 'name=The Subscriber' \
  -d 'l=eb420c55-4cfb-4972-92ba-c93c34ba475d'
```
</RequestExample>

<Note>For form requests, use `l` for multiple lists instead of `list_uuids`.</Note>

<ResponseExample>
```json
{
  "data": true
}
```
</ResponseExample>

## Modify List Memberships

<api-endpoint>
  <endpoint>
    <method>PUT</method>
    <path>/api/subscribers/lists</path>
  </endpoint>
</api-endpoint>

Modify subscriber list memberships.

### Body Parameters

<ParamField body="ids" type="number[]" required>
  Array of subscriber IDs to be modified.
</ParamField>

<ParamField body="action" type="string" required>
  Action to be applied: `add`, `remove`, or `unsubscribe`.
</ParamField>

<ParamField body="target_list_ids" type="number[]" required>
  Array of list IDs to be modified.
</ParamField>

<ParamField body="status" type="string">
  Required for `add` action. Subscriber status: `confirmed`, `unconfirmed`, or `unsubscribed`.
</ParamField>

<RequestExample>
```bash cURL
curl -u 'api_username:access_token' -X PUT 'http://localhost:9000/api/subscribers/lists' \
  -H 'Content-Type: application/json' \
  --data-raw '{
    "ids": [1, 2, 3],
    "action": "add",
    "target_list_ids": [4, 5, 6],
    "status": "confirmed"
  }'
```
</RequestExample>

<ResponseExample>
```json
{
    "data": true
}
```
</ResponseExample>

</RequestExample>

```json
    "data": true
}
```

## Error Responses

All endpoints return standard HTTP status codes:

- `200 OK` - Request successful
- `400 Bad Request` - Invalid parameters
- `401 Unauthorized` - Missing or invalid authentication
- `404 Not Found` - Subscriber not found
- `500 Internal Server Error` - Server error
