title: Bounces API
description: API endpoints for managing email bounce records in Listmonk

Manage email bounce records to track and handle delivery failures. Bounces can be hard (permanent) or soft (temporary) and help maintain list health.

## Get Bounces

<api-endpoint>
  <endpoint>
    <method>GET</method>
    <path>/api/bounces</path>
  </endpoint>
</api-endpoint>

Retrieve bounce records with optional filtering and pagination.

### Query Parameters

<ParamField query="campaign_id" type="number">
  Bounce record retrieval for particular campaign ID.
</ParamField>

<ParamField query="page" type="number">
  Page number for pagination.
</ParamField>

<ParamField query="per_page" type="number">
  Results per page. Set to 'all' to return all results.
</ParamField>

<ParamField query="source" type="string">
  Filter by bounce source.
</ParamField>

<ParamField query="order_by" type="string">
  Fields by which bounce records are ordered. Options: `email`, `campaign_name`, `source`, `created_at`.
</ParamField>

<ParamField query="order" type="string">
  Sorts the result. Options: `asc`, `desc`.
</ParamField>

### Response

<ResponseField name="data" type="object">
  <ResponseField name="results" type="array">
    Array of bounce objects.
    <ResponseField name="id" type="number">
      Bounce record's unique ID.
    </ResponseField>
    <ResponseField name="type" type="string">
      Bounce type: `hard`, `soft`.
    </ResponseField>
    <ResponseField name="source" type="string">
      Bounce source identifier.
    </ResponseField>
    <ResponseField name="meta" type="object">
      Additional metadata about the bounce.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      Timestamp when bounce was recorded.
    </ResponseField>
    <ResponseField name="email" type="string">
      Email address that bounced.
    </ResponseField>
    <ResponseField name="subscriber_uuid" type="string">
      Subscriber's UUID.
    </ResponseField>
    <ResponseField name="subscriber_id" type="number">
      Subscriber's ID.
    </ResponseField>
    <ResponseField name="campaign" type="object">
      Campaign information.
      <ResponseField name="id" type="number">
        Campaign ID.
      </ResponseField>
      <ResponseField name="name" type="string">
        Campaign name.
      </ResponseField>
    </ResponseField>
  </ResponseField>
  <ResponseField name="total" type="number">
    Total number of bounce records.
  </ResponseField>
  <ResponseField name="page" type="number">
    Current page number.
  </ResponseField>
  <ResponseField name="per_page" type="number">
    Results per page.
  </ResponseField>
</ResponseField>

## Bounce Types

### Hard Bounces
Permanent delivery failures that indicate the email address is invalid, doesn't exist, or the domain is not accepting mail. Subscribers with hard bounces should be removed from your lists.

Examples:
- Email address doesn't exist
- Domain doesn't exist
- Mailbox is disabled

### Soft Bounces
Temporary delivery failures that may resolve themselves. After multiple soft bounces, an address may be treated as a hard bounce.

Examples:
- Mailbox is full
- Server temporarily unavailable
- Message too large

## Bounce Handling

Listmonk automatically processes bounces when configured with:

1. **SMTP Bounce Processing**: Configure bounce handling in your SMTP settings
2. **Webhook Integration**: Receive bounce notifications from your email service provider
3. **API Integration**: Programmatically record bounces via the API

## Managing Subscriber Bounces

You can also manage bounces at the subscriber level:

- `GET /api/subscribers/{subscriber_id}/bounces` - View a subscriber's bounce records
- `DELETE /api/subscribers/{subscriber_id}/bounces` - Clear a subscriber's bounce records

See the [Subscribers API](/api/subscribers) documentation for more details.

## Error Responses

All endpoints return standard HTTP status codes:

- `200 OK` - Request successful
- `400 Bad Request` - Invalid parameters
- `401 Unauthorized` - Missing or invalid authentication
- `404 Not Found` - Bounce record not found
- `500 Internal Server Error` - Server error
